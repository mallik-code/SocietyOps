Code generation rules:

1. Never generate code without:
   - tests
   - migrations
   - logging

2. Enforce:
   - separation of concerns
   - dependency injection
   - error handling

3. Testing:
   - include edge cases
   - include failure scenarios

4. Database:
   - use migration tools only
   - ensure backward compatibility

5. Output format:
   - modular code
   - no monolithic files
